### 插值语法
```bash
{{xxx}}中，xxx可以是data中的所有属性，也可以是js表达式(一个表达式会产生一个值，可以放在任何一个需要值的地方),如:
(1). a
(2). a+b
(3). demo(1)
(4). x === y ? 'a' : 'b'
```
### 指令语法

#### v-text
```html
主要用于更新textContent，等同于js的text属性
<span v-text="msg"></span>
等价于
<span>{{msg}}</span>
```

#### v-html 输出html代码
```html
双大括号的方式会将数据解释为纯文本，而非HTML。为了输出真正的HTML，可以用v-html指令。等同于js的innerHtml属性
<div v-html="rawHtml"></div>
rawHtml的内容将会被渲染到div标签内
```

#### v-pre
```html
v-pre主要用来跳过这个元素和它的子元素编译过程。可以用来显示原始的Mustache标签。跳过大量没有指令的节点加快编译。
<span v-pre>{{message}}</span>  // 不编译
原样输出{{message}}这个字符串
```

#### v-cloak
```html
这个指令用来保持在元素上,直到关联实例结束时进行编译(解决页面加载时的闪烁问题)
<div id="app" v-cloak>
    <div>
        {{message}}
    </div>
</div>
<script type="text/javascript">
    new Vue({
      el:'#app',
      data:{
        message:'hello world'
      }
    })
</script>
不加v-cloak时，页面加载会先显示
<div>
    {{message}}
</div>
再显示
<div>
    hello world
</div>
即闪烁现象
```

```html
v-once关联的实例，只会渲染一次。之后的重新渲染，实例及其所有的子节点将被视为静态内容跳过，这可以用于优化更新性能。

```

#### v-if v-else-if v-else
```html
v-if可以实现条件渲染，Vue会根据表达式的值的真假条件来渲染元素。如果v-if的值是false，则这个元素被销毁，不在dom中
<div id="app">
    <p v-if="ok">ok ok ok</p>
    <p v-else-if="yes">yes  yes  yes</p>
    <p v-else>no no no</p>
</div>

<script>
    const v = new Vue({
        el: "#app",
        data: {
            ok: false,
            yes: false
        }
    })
</script>
```

#### v-show
```html
也是用于根据条件展示元素。和v-if不同的是，v-show的元素会始终被渲染并保存在dom中，它只是简单的切换css的dispaly属性。
<div id="app">
    <p v-show="ok">ok ok ok</p>
</div>

<script>
    const v = new Vue({
        el: "#app",
        data: {
            ok: false,
        }
    })
</script>

注意：
v-if有更高的切换开销
v-show有更高的初始渲染开销。
因此，如果要非常频繁的切换，则使用v-show较好；如果在运行时条件不太可能改变，则v-if较好
```
#### v-for
```html
用v-for指令根据遍历数组来进行渲染
<div id="app">
    <div v-for="(item,index) in items"> //使用in遍历，index是项的索引
        <ul>
            <li>{{index}}:{{item}}</li>
        </ul>
    </div>
    <div v-for="item of items"> //使用of遍历
        <ul>
            <li>{{ item }}</li>
        </ul>
    </div>
</div>

<script>
    const v = new Vue({
        el: "#app",
        data: {
            items: [
                {text: "aaa"},
                {text1: "bbb"}
            ]
        }
    })
</script>
注意：
当v-for和v-if同处于一个节点时，v-for的优先级比v-if更高。这意味着v-if将运行在每个v-for循环中
```

#### v-bind 是否使用属性
```html
v-bind用来动态的绑定一个或者多个特性。没有参数时，可以绑定到一个包含键值对的对象。常用于动态绑定class和style。以及href等。
简写为一个冒号:

<!-- 完整语法 -->
<a v-bind:href="url"></a>
<!-- 缩写 -->
<a :href="url"></a>
```
#### v-on 事件监听

```html
<!-- 完整语法 -->
<a v-on:click="doSomething"></a>
<!-- 缩写 -->
<a @click="doSomething"></a>
```

#### computed和methods 计算属性
我们可以使用 methods 来替代 computed，效果上两个都是一样的，但是 computed 是基于它的依赖缓存，只有相关依赖发生改变时才会重新取值。而使用 methods ，在重新渲染的时候，函数总会重新调用执行
```html
<!DOCTYPE html>
<html>
<head>
    <meta charset="utf-8">
    <title>测试</title>
</head>
<script src="static/vue.js"></script>
<body>
<div id="app">
    <p>原始字符串{{ message }}</p>
    <p>计算-反转字符串{{ reverseMessage }}</p>
    <p>方法-反转字符串{{ reverseMessage2() }}</p>
</div>
<div id="app1">
    <p>{{ message }}</p>
</div>
<script>
    var vm = new Vue({
        el: '#app',
        data: {
            message: 'Runoob'
        },
        computed: {
            reverseMessage: function () {
                return this.message.split('').reverse().join('')
            }
        },
        methods: {
            reverseMessage2: function () {
                return this.message.split('').reverse().join('')
            }
        }
    })

    var vm1 = new Vue({
        el: "#app1",
        data: {
            message: 'qwertyuiop',
        },
        computed: {
            site: {
                get: function () {
                    return this.message
                },
                set: function (value) {
                    this.message = value
                }
            }
        }
    })
    vm1.message = "fsdfadsfdswafgdsg"
</script>
</body>
```
